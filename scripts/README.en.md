# Scripts README

This directory contains helper scripts for data inspection and preprocessing.

## Requirements

- Python 3.8+ (standard library only, no third-party dependencies)
- Run commands from the repository root with `python3 scripts/...`

## Scripts

### `compare_starts.py`

Purpose:
- Compare start coordinates between TXT output from the original C implementation and CSV output from Rust.
- Column 2 in TXT is 0-based; this script converts it to 1-based before comparison.

Inputs:
- `txt`: TXT file generated by the C implementation
- `csv`: CSV file generated by the Rust implementation

Arguments:
- `--limit`: Maximum number of mismatches to print (default: `20`)

Examples:

```bash
python3 scripts/compare_starts.py ./old_impl.txt ./rust_impl.csv
python3 scripts/compare_starts.py ./old_impl.txt ./rust_impl.csv --limit 50
```

Output:
- Prints row counts for both files and the mismatch count.
- For each mismatch, prints the current row plus previous/next neighbors to help locate offset/grouping differences.

### `ncbi_region_split.py`

Purpose:
- Batch-read NCBI `*_genomic.gff(.gz)` and `*_genomic.fna(.gz)` files.
- Export all `feature=region` records from GFF into CSV.
- Split FNA by `seqid` and write `seqid.fna`.

Input directory convention:
- Each first-level subdirectory under `--input-root` is treated as one assembly.
- Each assembly directory should contain exactly one of each:
  - `*_genomic.gff` or `*_genomic.gff.gz`
  - `*_genomic.fna` or `*_genomic.fna.gz`

Arguments:
- `--input-root`: Assembly root directory (required)
- `--output-root`: Output root directory (required)
- `--on-mismatch`: `warn_skip` (default) or `strict`
- `--summary-name`: Summary CSV filename (default: `regions.all.csv`)

Example:

```bash
python3 scripts/ncbi_region_split.py \
  --input-root /path/to/refseq \
  --output-root /path/to/output
```

Strict mode (returns non-zero on missing/length mismatch):

```bash
python3 scripts/ncbi_region_split.py \
  --input-root /path/to/refseq \
  --output-root /path/to/output \
  --on-mismatch strict
```

Outputs:
- `output-root/<assembly_id>/regions.csv`
- `output-root/<assembly_id>/fna_split/<seqid>.fna`
- `output-root/<summary-name>`

Main fields in `regions.csv` / summary:
- `assembly_id, seqid, region_start, region_end, region_length, source, strand`
- `genome, mol_type, is_circular, name, strain, plasmid_name, taxon_id`
- `fna_header, fna_length, length_match, status, attributes_json`

`status` values:
- `ok`: `seqid` exists in FNA and length matches; split file was generated
- `missing_in_fna`: `seqid` exists in GFF but not in FNA
- `length_mismatch`: `region_length` does not match actual FNA sequence length
